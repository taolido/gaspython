# 講座全体設計

## 1. 講座タイトル

**正式タイトル:**
> **「挫折しない・壊さないPython入門」**
> ─ AIと一緒に学ぶ、GASの次のステップ

**サブタイトル案:**
> 環境構築で詰まらない。依存関係で壊さない。AIが隣にいるから怖くない。

---

## 2. 企画背景

### GAS講座の成功要因分析

| 成功要因 | 内容 |
|----------|------|
| 情報源の質 | シンギュラリティ・ラボ等の体系的コンテンツ |
| 明確なターゲット | 非エンジニアの業務効率化層 |
| 需要の明確さ | Google Workspace自動化ニーズ |
| 市場の隙間 | 日本語で体系的なGAS講座が少なかった |

### Python講座での対応

| GAS成功要因 | Python講座での対応 |
|------------|-------------------|
| 情報源の質が高い | 複数の良質ソース組み合わせ + 独自視点 |
| 非エンジニア向け | Google Colab + AI活用で敷居下げ |
| 需要明確（業務効率化） | 「GASで出来ないこと」特化 |
| 日本語コンテンツの隙間 | **「GASからPython移行ガイド」が少ない** |

---

## 3. 4本柱（講座の売り）

### 柱1: 環境構築で挫折させない

- Google Colabで即スタート
- ローカル環境は「必要になったら」
- 環境構築の説明は第4章まで後回し

### 柱2: AIを学習パートナーにする

- Claude/ChatGPTの正しい使い方（丸投げNG）
- エラー対処の型（聞き方テンプレート）
- 「なぜそうなるか」を聞く習慣化
- 再現可能なメソッドとして体系化

### 柱3: 目に見える成果を早期に出す

- 第3章で実用ツール完成
- 第4章でexe化して配布可能に
- 「作って終わり」ではなく「使ってもらえる」まで

### 柱4: 環境を壊さない作法を教える

- venvは「保険」ではなく「作法」
- グローバルインストール禁止
- 「壊れたら直さない、捨てて作り直す」思想
- 依存関係の基礎知識

## 4. ターゲット設定

### メインターゲット

| 属性 | 説明 |
|------|------|
| GAS講座受講者 | 既存顧客、アップセル対象 |
| 業務効率化に興味がある非エンジニア | Excel/スプレッドシートヘビーユーザー |

### サブターゲット

| 属性 | 説明 |
|------|------|
| VBA経験者 | 次のステップを探している |
| Python再入門者 | 過去に環境構築や情報過多で挫折 |
| AIコード生成利用者 | 生成されたコードを理解したい |

### ターゲットの課題

| 課題 | 本講座での解決策 |
|------|------------------|
| 環境構築が面倒 | Google Colab + AI活用 |
| 学習範囲が不明確 | GASで出来ないことに焦点 |
| AIツール選択に迷う | 学習パートナーとしての活用法 |
| 情報過多 | 業務効率化に特化 |
| 環境を壊してしまう | venv必須の作法を教える |

## 5. 他講座との差別化

| 観点 | 既存Python講座 | 本講座 |
|------|---------------|--------|
| 環境構築 | 最初に詳しく解説 | **後回し**（Colabで即開始） |
| AI活用 | 補助的に紹介 | **メソッドとして体系化** |
| 対象者 | 完全初心者 or エンジニア志望 | **GAS経験者・再入門者** |
| ゴール | Python習得 | **「使えるツールを作る」** |
| 挫折対策 | モチベーション系Tips | **具体的な仕組み（AI家庭教師）** |
| 環境管理 | venvは「あれば便利」程度 | **venvは「必須作法」** |

## 7. 技術方針

### OS対応

- **Windows優先**
- Mac補足（第4章で対応表を掲載）

### 環境

| 章 | 環境 | 理由 |
|----|------|------|
| 第0-3章 | Google Colab | 壊しようがない、即開始可能 |
| 第4-5章 | ローカル | exe化、定期実行のため |

### 環境管理ツール

- **venv推奨**（Python標準）
- **Anaconda非推奨**（初心者には複雑、pipとの混在問題）

## 8. 講座形式（検討事項）

| 形式 | メリット | デメリット |
|------|----------|------------|
| テキスト | 参照しやすい、更新容易 | 動画より伝わりにくい部分あり |
| 動画 | 操作が伝わりやすい | 更新コスト高、OS別に収録必要 |
| ハンズオン | 実践的 | 準備コスト高 |

**推奨:** テキスト + 要所で動画（環境構築など）
