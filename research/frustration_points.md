## 3. 環境構築の具体的な問題

### Windowsでの問題

- PATHの設定ミス
- PowerShellの実行ポリシー
- 複数Pythonバージョンの混在

### Macでの問題

- M1/M2チップでの互換性問題（一部改善）
- システムPythonとの混在
- Homebrewとの組み合わせ

### 共通の問題

- Anacondaとpipの混在による環境破壊
- グローバルインストールによる汚染
- 仮想環境の概念が理解できない

## 4. 依存関係エラーの問題

### よくあるエラー

```
ERROR: pip's dependency resolver does not currently take into account 
all the packages that are installed.
```

### 原因パターン

1. **グローバル環境の汚染**
   - いろいろなプロジェクトで`pip install`を繰り返す
   - バージョンが衝突

2. **Anacondaとpipの混在**
   - `conda install`と`pip install`を混ぜる
   - 依存関係の追跡が崩れる

3. **バージョン固定なしのインストール**
   - `pip install package`で最新版を入れる
   - 他パッケージとの互換性が崩れる

### 対策

- プロジェクトごとにvenvを作る
- `requirements.txt`でバージョンを固定
- 壊れたら直さない、作り直す

## 5. GAS経験者特有の問題

### GASとの違いによる混乱

| GAS | Python | 混乱ポイント |
|-----|--------|-------------|
| 環境構築不要 | 必要 | 「なぜこんなに面倒？」 |
| ブラウザで完結 | ローカル実行 | ファイル操作の概念 |
| Google連携が簡単 | API設定が必要 | 認証の複雑さ |
| 6分制限あり | 制限なし | 逆に自由すぎる |
| 共有が簡単 | exe化が必要 | 配布の手間 |

### GAS経験者のアドバンテージ

- 変数、関数、ループの概念は理解済み
- 「自動化の楽しさ」を知っている
- エラー対処の基本姿勢がある
